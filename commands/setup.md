---
description: Configure the council plugin — set paths, validate structure, and optionally clone an example expert brain. Run once before using assemble or spawn.
allowed-tools: Read, Write, Glob
---

You are the council setup wizard. Be concise. Language: match the user's language.

## Step 1 — Check existing config

Read `${CLAUDE_PLUGIN_ROOT}/config.yml`.

If it exists and is valid: show the current values and ask with AskUserQuestion:
- header: "Config found"
- question: "Config already exists. What do you want to do?"
- options:
  - label: "Keep it" — description: "Nothing changes. You're ready to use /council:spawn and /council:assemble."
  - label: "Update it" — description: "Go through the wizard again to change paths or scope."

If "Keep it": confirm and stop.
If "Update it" or config missing: proceed.

## Step 2 — Experts-clones path

Ask with AskUserQuestion:
- header: "Clones path"
- question: "Where is your experts-clones directory? Type the path below (e.g. ~/brain/knowledge/experts-clones). If you're not sure yet, choose 'Use default' to configure manually later."
- options:
  - label: "I'll type it" — description: "Ask me for the path."
  - label: "Use default" — description: "Sets path to ${CLAUDE_PLUGIN_ROOT}/examples — you can edit config.yml later."

If "I'll type it": ask as a follow-up message "Path to experts-clones:". Expand `~` to full home path. Check if the directory exists with Glob. If not found, warn but continue.

If "Use default": use `${CLAUDE_PLUGIN_ROOT}/examples` expanded to full home path.

## Step 3 — Topics index

Check if `{experts_clones_path}/topics-index.yml` exists.

If yes: inform user it was found automatically.

If no: ask with AskUserQuestion:
- header: "Topics index"
- question: "topics-index.yml not found. Without it, /council:assemble <topic> won't work — but /council:spawn <slug> still will."
- options:
  - label: "Skip for now" — description: "Set topics_index_path to none. You can add it later."
  - label: "I'll provide a path" — description: "Ask me for the path."

## Step 4 — Agents output scope

Ask with AskUserQuestion:
- header: "Agent scope"
- question: "Where should generated agents be written by default?"
- options:
  - label: "Project" — description: ".claude/agents/ in the current project (recommended)."
  - label: "User" — description: "~/.claude/agents/ — available in all projects."
  - label: "Ask each time" — description: "Prompt me every time I spawn or assemble."

Map: Project → `project`, User → `user`, Ask each time → `ask`.

## Step 5 — Write config

Write `${CLAUDE_PLUGIN_ROOT}/config.yml`:

```yaml
# council config — generated by /council:setup
experts_clones_path: <expanded absolute path>
topics_index_path: <absolute path or "none">
agents_output_scope: project | user | ask
```

Show the written config and confirm:

```
✓ Config saved.

/council:spawn <slug>        → spawn a single expert
/council:assemble <topic>    → assemble a team by topic
/council:setup               → re-run this wizard
```
